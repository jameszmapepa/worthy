# Security Policy

## Supported versions

worthy is pre-1.0; only the latest release receives security fixes.

| Version | Supported |
|---|---|
| 0.1.x   | yes       |
| < 0.1   | no        |

## Reporting a vulnerability

Please report security issues privately rather than opening a public issue.

- Preferred: open a [private security advisory](https://github.com/jameszmapepa/worthy/security/advisories/new).
- Or email **jameszmapepa@gmail.com**.

Include the version, steps to reproduce, and the impact you observed. Expect an
initial response within a few days.

worthy makes only read-only, unauthenticated GitHub API calls and stores no
credentials, so its attack surface is small — but reports are still welcome.
